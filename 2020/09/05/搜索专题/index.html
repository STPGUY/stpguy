深度优先搜索(Depth First Search)

优势：

  • 适合状态不易储存的情况
  • 有时候对子问题的依赖很强
  • 符合人类的思考习惯

劣势：

  • 很容易超时
  • 很容易爆栈

宽度优先搜索(Breadth First Search （Baidu First Search）)

优势：

  • 适合状态容易储存的问题
  • 不容易爆栈（搜索深度远大于$\text{DFS}$）
  • 能够将父状态继承给子状态
  • 可以跑最短路啊，各种最优化问题

劣势：

  • 可能产生大量的无用状态导致$MLE$
  • 对某些数据结构的要求有点高
